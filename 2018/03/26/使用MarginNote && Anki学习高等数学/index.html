关于舍本逐末的问题

很多人说这样子，舍本逐末，花费过多时间。
我想说的是，那个，你们的思路弄反了。

不是为了制作卡片才这么弄，mn之所以好用，是因为它让模拟了我们现实中的需求，让我们在软件中得以实现，让原本的操作更加便捷
不是为了制作卡片方才如此阅读一本书，而是我们需要这样深入阅读一本书，才使用mn来阅读这本书。
而卡片导入anki，只是一种锦上添花的，另人惊艳的快捷操作。

这让我想起了，以前的我想管理时间，网络上大谈特谈着各种软件，最为著名的就是Omnifocus,但是用过后你会发现，你的生活里是完全不需要到如此复杂的软件的，自己的生活里就是一个简单的三点一线的学生生活，亦或是简单的上下班而已。

在最后的我，时间管理上用 滴答清单 和 番茄土豆 就实现了时间的管理
诚然，这不是软广，哈哈！这两个软件我都不是会员，又真心是觉得好用，推荐下
我想说明的是什么呢，就是当你觉得这么做是浪费时间的时候，只是说明：
  1. 你平时的阅读习惯让你不会觉得mn是刚需
  2. 还有不是每一本书都需要这样来深度学习的

关于highlight

每次选高亮一个地方确实都会生成学习卡，可是学习卡里面已经有很清晰的逻辑了分类了
像平时做思维导图，我们需要一个这样的标题，但是这图片明显是不需要背诵的
所以，点那个标签过滤就可以了，就会自动过滤了呀！

当然，还有颜色的过滤，这个更高级了，其实，大多时候，我们把要背诵的自己加个标签就够了。
我感觉问这个问题，可能是不知道怎么做卡片吧，我分享下我平时做卡片的习惯。当然，只是我喜欢的方式，你可以参考下。
  1. 我会把要学这的一章的大纲先列出来，即把标题都剪切下来，形成了这个图。
  2. 在每小节里面，即图中的 7.3 关系的，里面有不同的小知识点做成卡片
比如这个表，是我们想要记忆的，我们把它截下来
然后点击
其实，可以说 你把这图上面的标题也直接截图下来，然后两个合成，合成两张图片的话，第一张会默认作为题目的，我手打标题的原因，是到时候可以索引。
4. 当卡片都做好了，我就会在下图的左边，圈选这些卡片
因为，按我的逻辑，基本要背诵的卡片都是在这本数的第四级上。所以直接圈一个距行就圈住我要背诵的卡片了。

关于如何挑选书籍

好像挺多人看的，有次有个女同学向我要一本书的资源，看了书名，我就说：这本书讲的不好，应该看这本。
然后就突然想到，怎么去挑选一本书，通过哪本书去学习对学习的影响实在是太过重要，同一个事物，不同的介绍，接受者的获取量会相差很多。
所以，思考着，写下来。
那就是 ：豆瓣标签
比如，还是拿高等数学为例
我就会在浏览器打
book.douban.com/tag/高等数学
这样，我们就知道，嗯，这本书是不错的呢！
book.douban.com/tag/你想要的书的类
其实，标签里运用的，最适合的是，你想了解一个领域的知识
比如，前端
book.douban.com/tag/前端
你看，如果有前端的同学，你们就会看到 那几本绕不过去的： js高级 ，es6 ，head first html 与 css
其实，说到这里了，已经不仅仅是简单的学习高等数学，其实我想表达的是，如何去学习一种学习方法，怎么在一个未知的领域入门。
如果，运用贯通的话，就会举一反三的。
还是以前端为例
看书，是一个不错的学习方法，但是，视频呢？
现在的学习资源获取实在是太简单了
且不说 诸如 学堂在线 幕课网 之类的 （哈哈，大学考试，不都是考前，用二倍速看教学视频吗？然后找一个什么也不会的同学，边学边教别人，两三天就可以复习（学）完一门科目了）
前端，这种怎么说呢，特别火的这种工作，就一定会有培训班的存在，有培训班，就默认有视频，有视频嘛，就会有百度云流出。。。。。额。。。。。
